# TS38.401 — NG-RAN; Architecture description

> 蒸馏页：常用结论 + 精确 clause 引用。深挖见 `wiki/sections.tsv`（spec=TS38.401）。
> 原始文档：[[38.401_NG-RAN_Architecture_j20]]（`raw_sources/specs/TS38.401_NG-RAN_Architecture/38.401_NG-RAN_Architecture_j20.md`）

- **职责**：NG-RAN 架构本体——节点、接口、CU/DU 切分、UE/会话关联与标识。gNB 研发的架构基准。

## 总体架构（§5）
- NG-RAN = **gNB** + **ng-eNB**，经 **NG** 接到 5GC（AMF/UPF），gNB 间经 **Xn**。
- **gNB 内部切分**：
  - **gNB-CU** ↔ **gNB-DU**：**F1** 接口（[[TS38.473_F1AP]]）；一个 DU 只连一个 CU，一个 CU 可连多个 DU。
  - **CU 再分 gNB-CU-CP / gNB-CU-UP**：**E1** 接口（控制面/用户面分离）。
  - CU-CP↔DU = F1-C，CU-UP↔DU = F1-U（GTP-U）。
- 用户面（§5.2）/ 控制面（§5.3）协议栈分别描述。

## 标识与关联（§6）
- NG-RAN 标识（§6.2）：gNB ID、NCGI、NR Cell Identity、TAC、AMF UE NGAP ID / RAN UE NGAP ID、gNB-CU/DU UE F1AP ID 等。
- UE 关联（§6.4）：跨 NG/Xn/F1/E1 的 UE 上下文关联。

## 接口与协议

|------|------|---------|
