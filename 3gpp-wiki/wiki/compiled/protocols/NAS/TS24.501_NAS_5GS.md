# TS24.501 — NAS protocol for 5G System (5GS)

> 蒸馏页：常用结论 + 精确 clause 引用。深挖见 `wiki/sections.tsv`（spec=TS24.501）。
> 原始文档：[[24.501_NAS_5GS_j62]]（`raw_sources/specs/TS24.501_NAS/24.501_NAS_5GS_j62.md`）

- **层**：NAS（CT WG1）；UE ↔ **AMF** 端到端，对 NG-RAN 透明（经 [[TS38.331_RRC]] SRB 承载、[[TS38.413_NGAP]] 透传）。
- 两大子层：**5GMM**（移动管理）与 **5GSM**（会话管理）。

## 5GMM（§5）
- 状态：**5GMM-DEREGISTERED ↔ 5GMM-REGISTERED**（§5.2）；子状态 5GMM-IDLE / 5GMM-CONNECTED。
- 过程类型：common（§5.4）/ specific（§5.5）/ connection management（§5.6）。

| 过程 | 类别 | 要点 |
|------|------|------|
| **Registration** | specific | initial / mobility / periodic / emergency；`RegistrationRequest`→`RegistrationAccept`→`RegistrationComplete` |
| Deregistration | specific | UE/NW 发起 |
| **Service Request** | CM | `ServiceRequest`（从 IDLE 恢复连接 / 上行数据） |
| Authentication | common | 5G AKA / EAP-AKA'；`AuthenticationRequest/Response` |
| Security Mode Control | common | `SecurityModeCommand/Complete`（激活 NAS 安全） |
| Identity / Configuration / Notification | common | `IdentityRequest`、GUTI 重分配等 |

- 标识：**5G-GUTI**、SUPI/SUCI、5G-S-TMSI、GUAMI。
- 安全：NAS 完整性保护 + 加密（与 AS 安全独立），密钥 K_AMF 派生 K_NASint/K_NASenc。

## 5GSM（§6）
- 网络请求（§6.3）/ UE 请求（§6.4）过程。
| 过程 | 要点 |
|------|------|
| **PDU Session Establishment** | `PDU Session Establishment Request`→Accept；分配 PDU session、QoS rules |
| PDU Session Modification | QoS / 参数修改 |
| PDU Session Release | 释放会话 |

- QoS：QoS flow（QFI/5QI）、QoS rules → 对接 [[TS37.324_SDAP]] 的 flow→DRB 映射。

## 消息格式（§8/§9）
NAS 消息头（EPD、安全头类型、消息类型）、IE 编码。
