# TS38.321 — Medium Access Control (MAC) protocol specification

> 蒸馏页：常用结论 + 精确 clause 引用。深挖见 `wiki/sections.tsv`（spec=TS38.321）按行号区间取原文切片。
> 原始文档：[[38.321_MAC_j20]]（`raw_sources/specs/TS38.321_MAC/38.321_MAC_j20.md`）

- **层**：MAC（RAN WG2）
- **职责**：逻辑信道复用/解复用、HARQ、调度相关（SR/BSR/PHR）、随机接入、DRX、复用与优先级处理。
- **上层**：经**逻辑信道**为 [[TS38.322_RLC]] 服务；**下层**：经**传输信道**用 [[PHY/README]]。
- 定时器/参数多由 RRC 经 `MAC-CellGroupConfig` / `LogicalChannelConfig` 配置（[[TS38.331_RRC]]）。

## 核心过程（§5）

| 过程 | clause | 要点 |
|------|--------|------|
| 随机接入 RA | §5.1 | 4-step / 2-step；竞争/非竞争 |
| 上行时间对齐 | §5.2 | `timeAlignmentTimer`、TA Command MAC CE |
| DL-SCH 数据 | §5.3 | DL HARQ（§5.3.2） |
| UL-SCH 数据 | §5.4 | UL grant、HARQ、复用、SR/BSR/PHR |
| DRX | §5.7 | 省电；onDuration/Inactivity/Retx 定时器 |
| SCell 激活/去激活 | §5.9 | SCell Act/Deact MAC CE |
| PDCP 复制激活 | §5.10 | Duplication Act/Deact MAC CE |
| BWP 操作 | §5.15 | `bwp-InactivityTimer` |
| 波束失败检测恢复 BFR | §5.17 | BFR MAC CE |
| 数据不活动监测 | §5.19 | `dataInactivityTimer` |
| 小数据传输 SDT | §5.27 | RRC_INACTIVE 下 CG-SDT/RA-SDT |

### 随机接入 RA（§5.1，4/2-step）
初始化 §5.1.1 → 资源选择 §5.1.2 → 前导发送(Msg1/MsgA) §5.1.3 → RAR 接收(Msg2/MsgB) §5.1.4 → 竞争解决(Msg3/Msg4) §5.1.5 → 完成 §5.1.6。
- 关键：`ra-ResponseWindow`、`ra-ContentionResolutionTimer`、`preambleTransMax`、backoff（§7.2）。

### UL-SCH 调度相关（§5.4）
| 子过程 | clause | 关键定时器/参数（RRC 配置） |
|--------|--------|------|
| HARQ operation | §5.4.2 | 异步、自适应；HARQ 进程数 |
| 复用与组装 LCP | §5.4.3 | 逻辑信道优先级（priority/PBR/bucketSizeDuration） |
| 调度请求 SR | §5.4.4 | `sr-ProhibitTimer`、`sr-TransMax` |
| 缓存状态上报 BSR | §5.4.5 | `periodicBSR-Timer`、`retxBSR-Timer`、`logicalChannelSR-DelayTimer` |
| 功率余量上报 PHR | §5.4.6 | `phr-PeriodicTimer`、`phr-ProhibitTimer`、`phr-Tx-PowerFactorChange` |

### DRX（§5.7，定时器均 RRC 配置）
`drx-onDurationTimer`、`drx-InactivityTimer`、`drx-RetransmissionTimerDL/UL`、`drx-HARQ-RTT-TimerDL/UL`、`drx-LongCycleStartOffset`、`drx-ShortCycle`、`drx-ShortCycleTimer`、`drx-SlotOffset`。

## RNTI（§7.1，Table 7.1-2 用途）
| RNTI | 用途 | 传输信道 |
|------|------|---------|
| C-RNTI | 动态调度单播 | DL-SCH / UL-SCH |
| Temporary C-RNTI | 竞争解决 / Msg3 | DL-SCH / UL-SCH |
| RA-RNTI / MSGB-RNTI | RAR（4-step / 2-step） | DL-SCH |
| CS-RNTI | 配置调度（SPS/CG 激活去激活重传） | DL/UL-SCH |
| P-RNTI | 寻呼+SI 变更通知 | PCH |
| SI-RNTI | 系统信息广播 | DL-SCH |
| TPC-PUCCH/PUSCH/SRS-RNTI | 功控 | — |
| G-RNTI / G-CS-RNTI | MBS 组播 PTM | DL-SCH |

> 完整 RNTI 取值范围见 §7.1 Table 7.1-1（0001–FFF2 为各类专用，FFFE=P-RNTI，FFFF=SI-RNTI）。

## 常用 MAC CE（§6.1.3，按 LCID 标识）
- 上行：Buffer Status Report（§6.1.3.1）、C-RNTI（§6.1.3.2）、Single/Multiple Entry PHR（§6.1.3.8/9）、Configured Grant Confirmation（§6.1.3.7）、BFR（§6.1.3.23）
- 下行：UE Contention Resolution Identity（§6.1.3.3）、Timing Advance Command（§6.1.3.4）、DRX Command / Long DRX Command（§6.1.3.5/6）、SCell Activation/Deactivation（§6.1.3.10）、Duplication Act/Deact（§6.1.3.11）、各类 TCI/SRS/CSI 激活去激活 CE

> §6.1.3 共定义 80+ 个 MAC CE（含 TCI/波束管理/IAB/NCR/定位等），按需查清单。
