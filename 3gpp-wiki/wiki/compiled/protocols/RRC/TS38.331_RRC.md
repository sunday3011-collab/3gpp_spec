# TS38.331 — Radio Resource Control (RRC) protocol specification

> 蒸馏页：常用结论 + 精确 clause 引用。深挖见 `wiki/sections.tsv`（spec=TS38.331）按行号区间取原文切片。
> 原始文档：[[38.331_RRC_j20]]（`raw_sources/specs/TS38.331_RRC/38.331_RRC_j20.md`）

- **层**：RRC（控制面核心，RAN WG2）
- **职责**（§1）：AS 层无线资源控制——连接建立/重配/释放、移动性、系统消息广播、测量配置、承载控制、UE 能力传输。
- **下层**：信令经 SRB 由 [[TS38.323_PDCP]] 承载（AS 安全激活后完整性保护+加密）。
- **上层**：透传 [[TS24.501_NAS_5GS]] 的 NAS 消息（可在 RRC 消息中捎带 piggyback）。

## RRC 状态与迁移（§4.2.1）

| 状态 | 关键特征 |
|------|---------|
| **RRC_IDLE** | 无 RRC 连接；UE 控制的小区(重)选择；监听 P-RNTI 短消息与 CN 寻呼(5G-S-TMSI)；可发 SI 请求 |
| **RRC_INACTIVE** | 已建连但挂起；UE 存 **Inactive AS context**；配置 **RNA**(RAN 通知区)；支持 **SDT**(小数据传输)；监听 CN 寻呼+RAN 寻呼(fullI-RNTI) |
| **RRC_CONNECTED** | UE 存 AS context；单播数据收发；可配 UE 专用 DRX；支持 MR-DC、CA、测量上报 |

- IDLE↔CONNECTED：`RRCSetup`/`RRCRelease`（§5.3.3 / §5.3.8）
- INACTIVE：进入由 `RRCRelease` 带 `suspendConfig`；恢复由 `RRCResumeRequest`/`RRCResume`（§5.3.13）

## 信令无线承载 SRB（§4.2.2）

| SRB | 逻辑信道 | 用途 |
|-----|---------|------|
| SRB0 | CCCH | RRC 消息（连接建立前） |
| SRB1 | DCCH | RRC 消息（可捎带 NAS）；SRB2 建立前的 NAS |
| SRB2 | DCCH | NAS 消息、含 logged measurement 的 RRC；优先级低于 SRB1；**AS 安全激活后**配置 |
| SRB3 | DCCH | (NG)EN-DC / NR-DC 下特定 RRC 消息 |
| SRB4/5/6 | DCCH | 应用层测量上报(4/5)、网侧数据采集 logged meas(6)；均 AS 安全后配置 |

- 拆分 SRB（split SRB）仅 SRB1/SRB2 支持。

## 核心过程（§5.3 连接控制）

| 过程 | clause | 关键消息 |
|------|--------|---------|
| 寻呼 Paging | §5.3.2 | `Paging` |
| 连接建立 | §5.3.3 | `RRCSetupRequest` → `RRCSetup` → `RRCSetupComplete`（或 `RRCReject`） |
| 初始 AS 安全激活 | §5.3.4 | `SecurityModeCommand` → `SecurityModeComplete` |
| 连接重配 | §5.3.5 | `RRCReconfiguration` → `RRCReconfigurationComplete`（含 `reconfigurationWithSync` 即 handover） |
| 连接重建 | §5.3.7 | `RRCReestablishmentRequest` → `RRCReestablishment` → `RRCReestablishmentComplete` |
| 连接释放 | §5.3.8 | `RRCRelease`（带 `suspendConfig` → INACTIVE） |
| 无线链路失败 RLF | §5.3.10 | 物理层问题→T310；RLF 后触发重建 §5.3.7 |
| 连接恢复 Resume | §5.3.13 | `RRCResumeRequest`/`RRCResumeRequest1` → `RRCResume` → `RRCResumeComplete` |
| 统一接入控制 UAC | §5.3.14 | 接入类别 barring 检查；T390 |

- 系统信息（§5.2）：MIB/SIB1 + 按需 SI（`DedicatedSIBRequest`，T350）。
- 测量（§5.5）：`measConfig` 配置 → 触发 `MeasurementReport`。

## UE 定时器（§7.1.1，关键项）

| Timer | 启动 | 超时动作 |
|-------|------|---------|
| **T300** | 发 `RRCSetupRequest` | 见 §5.3.3.7（建立失败处理） |
| **T301** | 发 `RRCReestablishmentRequest` | 进入 RRC_IDLE |
| **T302** | 收 `RRCReject`/`RRCRelease(waitTime)` | barring 解除通知 §5.3.14.4 |
| **T304** | 收含 `reconfigurationWithSync` 的 `RRCReconfiguration`（即 HO/切换） | HO 失败→发起重建；SCG 则发 SCG failure info §5.7.3 |
| **T310** | 检测到 SpCell 物理层问题（N310 个连续 out-of-sync） | 安全未激活→IDLE；否则发起 MCG failure info / 重建 §5.3.7 |
| **T311** | 发起 RRC 重建 | 选到合适 NR 小区→否则进入 IDLE |
| **T312** | 配置后触发测量上报且 T310 运行中 | 发起 MCG failure info / 重建 |
| **T316** | 发 `MCGFailureInformation` | 见 §5.7.3b.5 |
| **T319 / T319a** | 发 `RRCResumeRequest`（a 为 SDT 场景） | 见 §5.3.13.5 |
| **T380** | 收 `RRCRelease` 中 `t380`（周期性 RNAU 计时） | 发起 RNA 更新 §5.3.13 |
| **T390** | 某接入类别 barring（每类别一实例） | 见 §5.3.14.4 |

> 注：还有大量 `T342`–`T346x`（UEAssistanceInformation 各类 preference 上报禁止计时器）、Sidelink 计时器（T400/T420/T421）、NTN 计时器（T430）等，按需查 §7.1.1。

## 常量（§7.3）
- **N310**：判定 SpCell "out-of-sync" 的最大连续指示数（触发 T310）。
- **N311**：判定 "in-sync" 的最大连续指示数（停止 T310）。

## 消息逻辑信道（§6.2）
- 下行：BCCH（MIB/SIB）、PCCH（Paging）、DL-CCCH（`RRCSetup`/`RRCReject`）、DL-DCCH（`RRCReconfiguration`/`RRCRelease`/`RRCReestablishment`…）
- 上行：UL-CCCH（`RRCSetupRequest`/`RRCResumeRequest`）、UL-DCCH（`RRCSetupComplete`/`MeasurementReport`/`UEAssistanceInformation`…）
