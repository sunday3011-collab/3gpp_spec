# TS38.323 — Packet Data Convergence Protocol (PDCP) specification

> 蒸馏页：常用结论 + 精确 clause 引用。深挖见 `wiki/sections.tsv`（spec=TS38.323）。
> 原始文档：[[38.323_PDCP_j10]]（`raw_sources/specs/TS38.323_PDCP/38.323_PDCP_j10.md`）

- **层**：PDCP（RAN WG2）
- **职责**：数据传递、SN 维护、头压缩、加密/完整性、重排序与按序递交、重复检测、复制。
- **上层**：经**无线承载**承载 [[TS38.331_RRC]] 信令(SRB)与用户数据(DRB，经 [[TS37.324_SDAP]])；**下层**：经 RLC channel 用 [[TS38.322_RLC]]。

## 功能（§4.4）
数据传递（U/C 面）、PDCP SN 维护、头压缩（**ROHC** / **EHC** / 上行 **UDC**）、加密/解密、完整性保护/校验、基于定时器的 SDU 丢弃、PDU Set / PSI 丢弃、split & DAPS 承载路由、**复制 duplication**、重排序与按序递交、乱序递交、重复丢弃。

## 核心过程

| 过程 | clause | 要点 |
|------|--------|------|
| 实体建立/重建/挂起/重配 | §5.1.1–5.1.5 | 重建时重置 ROHC/EHC/UDC、重设状态变量、重传未确认 SDU |
| 数据传输（发/收） | §5.2.1 / §5.2.2 | 维护 COUNT、加密、（DRB）头压缩；接收侧重排序 |
| SDU 丢弃 | §5.3 | `discardTimer` 超时丢弃 |
| 状态上报 | §5.4 | PDCP Status Report（AM DRB / 重建/数据恢复时） |
| 数据恢复 | §5.5 | AM DRB 重传未确认 PDU |
| 头压缩 ROHC | §5.7 | RFC 3095/4815；profile 配置 |
| 加密/解密 | §5.8 | 算法+密钥由上层(RRC)提供 |
| 完整性保护/校验 | §5.9 | **SRB 强制**；DRB 可配 |
| PDCP 复制 | §5.11 | 激活/去激活（MAC CE，§5.11.1）、重复 PDU 丢弃 |

## 安全（§5.8 / §5.9）
- 加密 + 完整性保护由 PDCP 执行；密钥/算法来自 [[TS38.331_RRC]]（AS 安全激活）。
- SRB 始终完整性保护+加密（AS 安全激活后）；DRB 完整性保护可选配置。

## 状态变量（§7.1）
- **COUNT** = HFN + PDCP SN（32-bit，0..2³²−1）。
- 发送：`TX_NEXT`（下一个待发 SDU 的 COUNT）。
- 接收：`RX_NEXT`（下一个期望接收）、`RX_DELIV`（首个未递交、仍等待的 COUNT）、`RX_REORD`（触发 t-Reordering 的 COUNT 之后值）。

## 常量与定时器（§7.2 / §7.3）

| 项 | 说明 | 引用 |
|----|------|------|
| `Window_Size` | 重排序窗口，由 `pdcp-SN-Size` 决定 | §7.2 |
| `discardTimer` | 发送侧，仅 DRB；收到上层 SDU 时启动，超时丢弃 | §7.3 / §5.2.1 |
| `discardTimerForLowImportance` | 发送侧，仅 DRB；低重要性 PDU Set | §7.3 |
| `t-Reordering` | 接收侧检测 PDCP Data PDU 丢失（每实体单实例） | §7.3 / §5.2.2 |

> SN 长度 `pdcp-SN-Size`：12 或 18 bit（由 RRC 配置）。定时器时长均由 RRC（TS38.331）配置。
